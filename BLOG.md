> 14,680 条中文网安技术问答，防御导向、零国家归因、零地缘内容，CC BY 4.0。
> 一行 `load_dataset` 即可用。这篇文章重点不是"我做了个数据集"，而是**它凭什么敢叫"纯净"**。

## 一、为什么又要一个网安数据集

做安全方向 LLM 微调的人都遇到过同一个尴尬：**中文、垂类、能直接用的安全问答数据，少得可怜**。

公开能找到的，大多是这几种：
- 英文为主，中文语料稀薄；
- 用大模型一把梭生成、没做任何清洗，重复、跑题、答案三句话就没了；
- 混进了大量"某国黑客组织""APT××"这类**带归因、带地缘立场**的内容——一旦你拿它去微调，模型就可能学会输出有政治风险的表述，在合规、教学、企业场景里根本不敢用。

这个数据集想解决的就是最后一点：**把"纯技术"和"可信赖"做到可被验证。**

## 三、重点来了：凭什么叫"纯净"——讲讲质检流水线

我不想用"人工精心校对"这种没法验证的话来描述质量。**我能给你的是可复现的机器保证。**

数据从生成到上线，过了三道关：

**1）规则化清洗流水线**
- 去除**完全重复**的问答；
- 删除**模板化填充**样本（那种"什么是 X 中的常见安全问题 N？"、答案凑到一百多字就结束的水货）；
- 删除**过短**答案；
- 同一个问题只保留**最完整的一份**答案。

**2）"污染词"过滤（这是和其他数据集最大的区别）**
用精准正则拦截**国家归因 / 命名 APT 组织 / 地缘政治**类内容，例如 `APT\d+`、"某国黑客组织"、命名威胁组织、"地缘政治"等。关键是**精准**——不会误伤 `CIA 三元组`、`MSSQL`、"APT 攻击技术（泛指）"这类正常技术术语。

**3）CI 自动校验（机器担保，不靠我嘴说）**
仓库挂了 GitHub Actions，每次提交自动跑 `scripts/check_dataset.py`，校验：
- 合法 JSON；
- 统一 `user`+`assistant` schema；
- 无重复问题；
- 无模板填充；
- 答案最短长度；
- 纯净度（上面那套污染词规则）。

**任何一条不过，CI 直接红灯。** 也就是说——你看到的绿标，就是"零非法 JSON / 零重复 / 零归因"的实时证明，不是我写在 README 里的口号。

## 四、诚实声明（我觉得这比吹牛更重要）

- 数据由**大语言模型生成**，不是人工逐条手写；
- 质量来自**上面的规则化流水线 + CI**，我**没有**做多模型交叉验证、也没做人工标注一致性评估——别的地方你可能见过这类说法，我这里不声称我没做的事；
- 它适合做**领域知识注入 / 风格对齐 / 冷启动**，但用于严肃训练前，请按你的场景自行抽检。

把丑话说在前面，是因为这个数据集的全部价值就建立在"可信"上。**夸大一句，整个就废了。**
